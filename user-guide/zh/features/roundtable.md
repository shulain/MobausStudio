# 圆桌会议

圆桌会议是 MobausStudio 的多 Agent 协作功能，让多个 AI 角色围绕一个主题展开讨论，从不同角度分析问题。

## 什么是圆桌会议？

想象一个会议室，多位专家围坐在圆桌旁讨论问题：
- 每位专家有不同的背景和视角
- 他们可以互相引用和回应观点
- 最终形成全面的分析和建议

圆桌会议就是让多个 AI 智能体扮演不同角色，模拟这种多角度讨论。

---

## 使用场景

### 产品决策
让「产品经理」「技术负责人」「用户体验设计师」共同讨论新功能方案。

### 技术方案评审
让「架构师」「后端专家」「安全专家」评估技术方案的可行性。

### 内容创作
让「编辑」「SEO 专家」「目标用户代表」讨论文章的改进方向。

### 问题分析
让不同领域的专家从各自角度分析复杂问题。

## 创建圆桌会议

### 第一步：进入编排模式

1. 在对话页面，点击顶部的「编排对话」按钮
2. 选择「圆桌会议」模式

### 第二步：设置讨论主题

输入要讨论的主题，例如：
- "如何提高网站的用户转化率？"
- "新产品应该优先开发哪些功能？"
- "如何优化系统的性能瓶颈？"

### 第三步：添加参与者

点击「添加参与者」，为每位参与者配置：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| Agent | 选择已创建的智能体 | 产品经理助手 |
| 角色 | 在本次讨论中的角色 | 产品经理 |

> 💡 **提示**：至少需要 2 个参与者，最多支持 6 个。

### 第四步：配置讨论规则（可选）

#### 发言模式

| 模式 | 说明 |
|------|------|
| 顺序发言 | Agent 按顺序轮流发言，每轮每人发言一次 |
| 自由发言 | 使用 @角色名 指定 Agent 发言，灵活控制讨论节奏 |

#### 讨论轮数

- **固定轮数**：设置 1-10 轮，讨论会在指定轮数后自动停止
- **不限制**：手动控制讨论进度，随时可以继续或结束

#### 其他选项

- **自动生成总结**：讨论结束后自动生成总结报告
- **允许引用他人观点**：Agent 可以引用和回应其他参与者的观点

### 第五步：添加背景信息（可选）

提供讨论的背景信息，帮助 Agent 更好地理解上下文：

```
我们是 B2B SaaS 产品，当前注册转化率 15%，
目标用户是企业 IT 管理员，
主要竞争对手是 XXX 和 YYY...
```

### 第六步：设置约束条件（可选）

限定讨论的方向和边界，避免 Agent 给出不切实际的建议：

```
- 预算不超过 5 万元
- 需要在 2 周内完成
- 不考虑重构整个架构
```

### 第七步：开始讨论

点击「开始讨论」，圆桌会议正式开始。

---

## 讨论过程

### 顺序发言模式

1. 系统会按顺序让每个参与者发言
2. 每轮结束后，可以点击「下一轮」继续
3. 达到设定轮数后，可以生成总结

### 自由发言模式

1. 在输入框中使用 `@角色名` 指定发言者
2. 例如：`@架构师 你觉得这个方案的可扩展性如何？`
3. 可以随时向任何参与者提问

### 中途干预

你可以随时：
- 提出新问题引导讨论方向
- 要求某位参与者详细解释
- 补充新的背景信息

---

## 生成总结

讨论结束后，点击「生成总结」按钮：

1. 系统会分析所有参与者的观点
2. 提取关键结论和建议
3. 生成结构化的总结报告

总结通常包含：
- 各方主要观点
- 达成的共识
- 存在的分歧
- 行动建议

## 使用技巧

### 1. 选择合适的 Agent

为不同角色选择专业领域匹配的 Agent：
- 技术讨论选择配置了技术相关技能的 Agent
- 产品讨论选择配置了产品分析技能的 Agent

### 2. 明确角色定位

给每个参与者设置清晰的角色，例如：
- 不要只写「专家」，而是写「后端架构专家」
- 不要只写「用户」，而是写「企业 IT 管理员」

### 3. 提供充足背景

背景信息越详细，讨论越有针对性：
- 当前状况和数据
- 已尝试过的方案
- 具体的约束条件

### 4. 适时引导

如果讨论偏离主题或陷入僵局：
- 提出新的问题引导方向
- 要求参与者回应特定观点
- 补充新的信息或约束

## 示例：产品功能讨论

### 配置

**主题**：我们的 SaaS 产品下个季度应该优先开发哪些功能？

**参与者**：
1. 产品经理 Agent → 角色：产品经理
2. 技术助手 Agent → 角色：技术负责人
3. 用户研究 Agent → 角色：用户体验设计师

**背景信息**：
```
- 当前 MAU 5000，付费转化率 8%
- 用户反馈最多的是「导出功能不完善」和「移动端体验差」
- 竞品最近上线了 AI 助手功能
```

**约束条件**：
```
- 开发资源：2 名后端 + 1 名前端
- 时间：3 个月
- 预算：20 万
```

### 讨论结果

通过多轮讨论，可能得出：
- 产品经理关注用户需求和市场竞争
- 技术负责人评估实现难度和资源需求
- 设计师关注用户体验和使用场景

最终形成优先级排序和实施建议。

## 常见问题

### 圆桌会议和普通对话有什么区别？

| 功能 | 普通对话 | 圆桌会议 |
|------|---------|---------|
| 参与者 | 1 个 AI | 多个 AI 角色 |
| 视角 | 单一视角 | 多角度分析 |
| 互动 | 用户与 AI | AI 之间也可互动 |
| 适用场景 | 问答、任务 | 讨论、决策 |

### 参与者必须是不同的 Agent 吗？

建议使用不同的 Agent，这样每个角色有不同的「人设」和专业背景。但也可以使用同一个 Agent 扮演不同角色。

### 讨论轮数设置多少合适？

- 简单问题：2-3 轮
- 复杂决策：5-8 轮
- 深度分析：可以不限制，根据需要继续
